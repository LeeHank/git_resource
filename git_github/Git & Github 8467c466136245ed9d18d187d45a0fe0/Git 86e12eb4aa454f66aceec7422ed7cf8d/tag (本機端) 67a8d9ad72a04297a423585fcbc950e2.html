tag (本機端)

  • 為啥要用 tag？
  • 因為做軟體開發時，都會有版本的需求，所以需要在commit上標上版本
  • 那這個概念，其實就是貼上貼紙，而我們之前學過 branch 和 HEAD 這兩種貼紙，現在要教的第三種貼紙，叫 tag 。
    • branch是個動態的貼紙，一路跟著commit走
    • HEAD也是個動態的貼紙，一路跟著branch走(所以也一路跟著commit走)
    • 而tag是個靜態的貼紙，貼上去就不會動了，那就可以標好這個commit所在的版本

0. 版本號

  • 寫版本號的時候，三個碼都是有意義的。
  • 舉例來說： 2.0.0 這個版本：
    • 第一個碼是主版號(major): 有大改，可能出現兼容性問題，就會改這個(e.g. 之前可以在IE上用，但新版本只能在chrome上用，那就會改這個碼)
    • 第二個碼是次版號(minor): 新功能加入時，會改這個
    • 第三個碼是修訂號(patch)，針對issue改bug時，就會加這邊的碼

1. 增加 tag

1.1. 增加輕量tag (不推薦，此tag不是git的tag物件)

# 先進到目前的 git 專案資料夾
cd {目前的 git 專案資料夾}

# 確認目前在哪個分支 (移動到你要下 tag 的 commit 的所在分支)
git checkout {你要下 tag 的 commit 所在的分支}

# 下 tag
git tag v1.0.0 # 對目前HEAD所在的commit，下tag
git tag v1.0.0 {SHA-1值} # 對之前的commit，下tag

1.2. 增加有註解的tag (推薦)

# 先進到目前的 git 專案資料夾
cd {目前的 git 專案資料夾}

# 確認目前在哪個分支 (移動到你要下 tag 的 commit 的所在分支)
git checkout {你要下 tag 的 commit 所在的分支}

# 下 tag
git tag v1.0.0 -a -m {"版本的註解寫在這拉"} # 對目前HEAD所在的commit，下tag
git tag v1.0.0 {SHA-1值} -a -m {"版本的註解寫在這拉"} # 對之前的commit，下tag

2. 檢查 tag

  • 我們可以從 .git資料夾 裡面，看tag
# 先進到目前的 git 專案資料夾
cd {目前的 git 專案資料夾}

# 查詢現有的 tag
git tag

# 在 .git/refs/tags下，可以看到你下過的 tags
ls .git/refs/tags # 可以看到你下過的tag。例如，會看到剛剛下過的 "v1.0.0" 這個 tag，他就會是一個檔案
cat .git/refs/tags/v1.0.0 # 看內容，會發現就是個SHA-1，此SHA-1就是你tag貼到的那個commit的SHA-1

# 如果是用 "有註解的tag"，那就會多存一個tag object，放在.git/objects下
git cat-file -t 此tag的SHA-1 #可以看到是tag物件
git cat-file -p 此tag的SHA-1 #可以看到剛剛下的註解("版本的註解喜在這拉")，誰下的，什麼時候下的，就像對commit物件的待遇)

3. 刪除 tag

git tag -d v1.0.0 # 刪除這個tag
  • 這邊提醒一下：刪除tag，只會刪掉 .git/refs/tags/ 下面的檔案，不會刪掉 .git/objects 裡面的tag物件，所以這些tag物件只會在之後資源回收時被清掉
